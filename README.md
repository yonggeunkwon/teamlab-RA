## 스터티 정기 회의

설명 방식

1. 각자 문제 풀이를 위한 접근 방식(or 개념) 설명
2. 기본 코드에 대한 설명
3. 기타(문제 풀이에 어려웠던 점, 구현하고자 했는데 실패한 방식, 추가적으로 개선한 점)

## PR 규칙 및 Commit Message 규칙

Pull Request

- [날짜] 이름
- ex) [20240104] yonggeun

Commit Message

- [문제 이름] 이름
- ex) [숨어있는 숫자의 덧셈 (2)] yonggeun
